# Public Visibility Claim Gate

Date: 2026 06 25

Status: local gate for public post, comment, outreach, and reply text. It protects the project from turning weak public signals into stronger claims than the evidence supports.

## Use this before public text

Use this gate before LinkedIn posts, LinkedIn comments, GitHub issue comments, outreach replies, public summaries, and README navigation text.

Run `make public_github_route_preflight` before naming live GitHub route state in public text.

## Gate rows

1. If the text mentions a live GitHub route, recheck the route before writing the claim.
2. If the text mentions a reply, call it a reply or acknowledgement only.
3. If the text mentions a public comment, call it one comment only.
4. If the text mentions open source work, do not imply merge, acceptance, review completion, or maintainer approval.
5. If the text mentions medical AI evaluation, keep it at safety evaluation or claim hygiene level.
6. If the text invites review, ask for one concrete objection or one weak sentence.

## Blocked claims

1. Endorsement.
2. Partnership.
3. Institutional support.
4. Clinical validation.
5. Clinical deployment.
6. Model superiority.
7. Score certification.
8. Patient data use.
9. Regulatory approval.
10. Terms or payment action.

## Allowed wording

1. A public comment was received.
2. A maintainer replied.
3. A route is open.
4. A local draft is ready.
5. A validator passed.
6. A reviewer can point to one weak sentence.

## Current action

Keep public visibility separate from route pressure. Drafts can ask for critique. They should not claim acceptance, review, endorsement, deployment readiness, or institutional backing.
