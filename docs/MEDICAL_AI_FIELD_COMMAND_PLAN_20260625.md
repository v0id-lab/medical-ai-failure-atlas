# Medical AI Field Command Plan

Date: 2026 06 25

Status: public field command plan and runnable synthetic eval path.

## Decision

Build the Turkish internal medicine safety evaluation lane as the fastest defensible path to visible medical AI authority.

## Position

A clinician led safety and source support layer for Turkish and English internal medicine scenarios, focused on missing context, escalation, medication safety, laboratory interpretation, source support, and claim hygiene.

## Current Field Read

### READ001

Claim: Global medical AI is crowded in model branding, general health chat, radiology, and benchmark leaderboards.

Evidence ids: SRC001, SRC006, SRC007, SRC008

Action: Avoid model race language. Build clinician review rows that model builders can reuse.

### READ002

Claim: The new credible edge is workflow evaluation under missing information, source support, escalation safety, and local clinical language.

Evidence ids: SRC003, SRC004, SRC005, SRC008, SRC009, SRC010

Action: Make every row name missing variables, dangerous reassurance, source gap, and safer wording.

### READ003

Claim: Türkiye has official health AI activity, decision support vendors, and radiology momentum, but a public internal medicine evaluation lane is still underbuilt.

Evidence ids: SRC011, SRC012, SRC013, SRC014, SRC016

Action: Lead with Turkish internal medicine safety evaluation rather than broad AI transformation claims.

## Two Agent Operating Model

### Agent one: Field radar

Job: Track FDA, WHO, IMDRF, HealthBench, MedGemma, MedHELM, AgentClinic, TÜYZE, KDS, NeyimVar, TÜBİTAK, and TEKNOFEST signals.

Output: One source checked weekly field note and one route owner action gate.

### Agent two: Eval factory

Job: Turn synthetic Turkish internal medicine scenarios into rows with escalation, missing variable, source support, and scoring gates.

Output: Validated JSONL rows, generated docs, and a maintainer ready issue pack.
