# Six Month Compressed Build Map

Date: 2026 06 25

## Six month outcomes

### 1. Severity rubric becomes the repo center

Evidence: A public spec defines severity grade, harm mechanism, missing variable, safer action, and reviewer state.

Risk: If the rubric is not central, the repo remains a pile of documents.

### 2. SourceCheckup becomes a public review lane

Evidence: Every claim review path asks what source support is missing and what evidence would change the statement.

Risk: If source support stays separate, the repo loses the bridge to general open source eval work.

### 3. Issue 154 becomes the public objection gateway

Evidence: Outside comments are routed into severity, source, wording, or claim hygiene work.

Risk: If comments are treated as validation, public credibility drops.

### 4. Turkish clinical AI path becomes readable

Evidence: A team can move from Turkish scenario to reviewer question to report wording gate without reading the whole repository.

Risk: If local language work is buried, the repo competes in the crowded global benchmark lane.

### 5. Open source eval identity stays visible

Evidence: lighteval, inspect ai, and future eval routes stay in the public route watchlist with honest states.

Risk: If medical AI hides open source AI work, the second equal identity disappears.

### 6. Public claims stay smaller than evidence

Evidence: README and validators block endorsement, validation, deployment, partner, institution, score, acceptance, and merge claims.

Risk: If claims outrun evidence, the repo becomes unsafe to share.
